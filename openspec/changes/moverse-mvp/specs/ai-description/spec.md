## ADDED Requirements

### Requirement: Generate a short item description from a photo

The system SHALL provide a `/describe` endpoint that accepts a base64 item photo and returns a short (≤ ~10 word) plain-text description naming the item, generated by a vision model. The endpoint SHALL keep provider secrets server-side (never on the device).

#### Scenario: Photo yields a short description

- **WHEN** the app posts a base64 photo of a packed item to `/describe`
- **THEN** the endpoint returns a short plain-text phrase naming the item (e.g. "Black winter boots"), with no preamble

#### Scenario: Secrets stay server-side

- **WHEN** the app calls `/describe`
- **THEN** the request does not contain the Anthropic API key; the key is held only by the proxy

#### Scenario: Upstream failure surfaces cleanly

- **WHEN** the vision provider call fails or times out
- **THEN** the endpoint returns an error response the app can present, and the app lets the user retry or type a description manually
